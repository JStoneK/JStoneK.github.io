执行完后，目录下会有两个文件夹：linux-6.6.48、buildroot-2024.02.6

3、Linux 内核裁剪

1、进入内核源码目录

2、加载 x86_64 架构默认配置make x86_64_defconfig

✅ 成功标志：终端显示 configuration written to .config

3、打开内核图形化配置界面

File systems → 回车

找到 The Extended 4 (ext4) filesystem → 按 N 取消（空框[]）

找到 Btrfs filesystem support → 按 Y 勾选（实心<*>）

按 Tab 选 <Save> → 回车保存 → 回车确认，这里不保存也可以，因为下一步退出（连按ESC）会提示是否保存已修改的内容，选择yes回车即可

连续按 ESC 退出配置界面

5、编译裁剪后的内核

（-j4用 4 线程编译，适配虚拟机配置，等待 5-10 分钟完成，这里你有几个核就用几，例如你是8个核，用make bzImage -j 8）

这里耐心等待即可

✅ 成功标志：最后提示 Kernel: arch/x86_64/boot/bzImage is ready

4、定制 Btrfs 根文件系统

1、进入 Buildroot 目录

2、打开配置界面

3、关键配置

1、Target options → Target Architecture 选 x86_64，回车进去有具体选项

2、Filesystem images → 勾选 btrfs root filesystem

3、System configuration → Default shell 选 bash

若出现以下问题，则没有显示看不到bash

按ESC回到 Buildroot 的主菜单。

用方向键选中 Target packages，按回车进入。

找到 Shells 选项，按回车进入。

找到 bash 选项，按Y键勾选（前面会出现<*>标记）。若没有无需管进行下一步

在 Target packages 的最顶部，找到Show packages that are also provided by busybox，按Y勾选，这样能显示更多相关选项，避免后续配置看不到。

此时我们可以看到了

4、进入Text editors and viewers

更改vim

5、进入Networking

选择openssh

6、更改下载源

选择download

选择第一个Primary download site

回车进入更换源

图中是科大源，清华源是更建议更换的（直接复制给的命令即可）

7进入 Kernel 菜单（主菜单里的 Kernel，按回车进去）

  1. 按空格勾选 [/*] Linux Kernel（不勾上的话，后面的配置选项不会显示）

  2. 找到 Kernel configuration 下面的选项（默认是个下拉菜单），按回车进入

  3. 选择

    Use the architecture default configuration

    这一步的意思是：让 Buildroot 自动使用 x86_64 架构的默认内核配置，不用你手动指定文件，彻底解决报错。

8、保存配置 → ESC 退出

一直按ESC，最后提示是否保存，yes即可

2、启动虚拟机（直接复制整条命令）

✅ 成功标志：进入命令行，系统正常启动！

✅ 关键参数说明：

  • -device edu：挂载实验要求的 edu 虚拟设备
  • -virtfs：配置 9p 文件共享，用于宿主机和虚拟机传输文件
  • -nographic：无图形模式，终端直接作为虚拟机控制台
